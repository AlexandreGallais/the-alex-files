## Service de gestion d’état

J'ai été confronté à une problématique : comment gérer les changements d'état dans mon application.

Le problème était :
- J'ai beaucoup de valeurs, reliées à des clés, qui sont mises à jour par un back-end.
- Ces valeurs sont mises à jour partiellement, à un certain cadencement.
- Je dois notifier mes features quand les clés qui les intéressent changent de valeur.
- Une feature doit pouvoir changer sa liste de clés en cours de route, sans avoir à se réabonner.

### Interfaces

J'ai donc besoin :
- D'une méthode de mise à jour des valeurs.
- D'une méthode d'abonnement aux changements.

#### Mettre à jour des données

```ts
public readonly patch = (patch: Entries<DataKey, DataValue>): void => {
  let hasChanges = false;

  // Je boucle sur toutes les valeurs du patch.
  for (const [key, value] of patch) {
    // Je ne mets à jour que les valeurs qui ont changé.
    if (Object.is(this.state.get(key), value)) {
      continue;
    }

    this.state.set(key, value);

    // J'ajoute la valeur aux patches des listeners abonnés à cette clé.
    for (const listener of this.listenersByKey.get(key) ?? []) {
      let pendingPatch = this.pendingPatchByListener.get(listener);

      if (!pendingPatch) {
        pendingPatch = new Map();

        this.pendingPatchByListener.set(listener, pendingPatch);
      }

      pendingPatch.set(key, value);
    }

    hasChanges = true;
  }

  if (!hasChanges) {
    return;
  }

  // S'il y a eu des changements, je demande la notification des listeners.
  this.scheduleFlush();
};
```

#### Notifier les listeners

Une des choses qui rend cette feature intéressante est l'utilisation de `queueMicrotask`.

Cela permet de regrouper les notifications des listeners dans une même microtask, une fois les appels synchrones à `patch` terminés.

Grâce à cela, je peux appeler plusieurs fois la méthode `patch` sans notifier les listeners à chaque appel, et sans dépendre d'un timer ou d'une règle arbitraire.

```ts
private isFlushScheduled = false;

private readonly scheduleFlush = (): void => {
  if (this.isFlushScheduled) {
    return;
  }

  this.isFlushScheduled = true;

  queueMicrotask(() => {
    for (const [key, signals] of this.signalsByKey) {
      for (const keySignal of signals) {
        keySignal.set(this.state.get(key));
      }
    }

    for (const [listener, pendingPatch] of this.pendingPatchByListener) {
      try {
        listener(pendingPatch);
      } catch {
        // Error handler
      }
    }

    this.pendingPatchByListener.clear();
    this.isFlushScheduled = false;
  });
};
```

#### Écouter les changements de valeur

Avec cette méthode, je peux écouter une liste de clés contenue dans un signal. Cela permet à la feature de ne pas se réabonner si elle doit changer de clés : elle change simplement son signal.

```ts
public readonly addDataListener = (
  keysSignal: Signal<IterableSource<DataKey>>,
  listener: DataListener,
): RemoveDataListener => {
  let previousKeys = new Set<DataKey>(keysSignal());

  this.addListenerToNewKeys(new Set<DataKey>(), previousKeys, listener);

  const ref = effect(() => {
    const nextKeys = new Set(keysSignal());

    this.removeListenerFromRemovedKeys(previousKeys, nextKeys, listener);
    this.addListenerToNewKeys(previousKeys, nextKeys, listener);

    previousKeys = nextKeys;
  });

  return () => {
    this.removeListenerFromAllKeys(previousKeys, listener);
    previousKeys.clear();
    ref.destroy();
  };
};
```

##### Création de signal

C'est la même idée que `addDataListener`, mais ici, on retourne un signal qui sera mis à jour quand la valeur de la clé change.

```ts
public readonly createDataSignal = (key: DataKey): SignalHandle => {
  const keySignal = signal(this.state.get(key));
  let keySignals = this.signalsByKey.get(key);

  if (!keySignals) {
    keySignals = new Set();

    this.signalsByKey.set(key, keySignals);
  }

  keySignals.add(keySignal);

  return {
    signal: keySignal.asReadonly(),
    destroy: (): void => {
      keySignals.delete(keySignal);

      if (keySignals.size !== 0) {
        return;
      }

      this.signalsByKey.delete(key);
    },
  };
};
```
